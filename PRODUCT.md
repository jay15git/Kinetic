# Product

## Register

brand

## Users

React developers evaluating or installing a scrub number input for design-tool-style UIs. They arrive from docs, registry links, or GitHub with a specific job: see the component behave, tune props, and copy install code.

## Product Purpose

Kinetic is a shadcn-registry **Scrub Number Input** for drag-to-scrub number input with animated digits, built on Base UI NumberField. The home page at `/` covers features (demo, gestures, API reference) and installation (Command, Manual). The demo at `/demo` is the settings playground with generated usage snippets. Success means a developer installs the component in under a minute with confidence in how it feels.

## Brand Personality

Precise, tactile, quiet. A finished instrument panel—not a marketing site. Voice: mechanical confidence, show-don't-tell, zero hype.

## Anti-references

Wide SaaS hero layouts, three-card feature grids, gradient backgrounds, oversized rounded cards, uppercase section kickers on every block, purple-blue AI styling, cream/warm body backgrounds, ghost-card border-plus-shadow combos.

## Design Principles

- Practice what you preach: the landing uses the same scrub ergonomics it sells.
- Show, don't tell: the live fields are the headline.
- Compact density over marketing drama: one narrow column, hairline rhythm, small type.
- Install path stays visible: command and props code are first-class, not buried.
- Motion is restrained and width-stable: no layout shift during digit animation.

## Accessibility & Inclusion

WCAG AA contrast on all body and control text. Full keyboard scrub/edit support demonstrated in the demo. Respect `prefers-reduced-motion` for page entrance and micro-interactions.
